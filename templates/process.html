Process

I design content systems and workflows that allow teams to deliver high-quality content efficiently, using automation and tooling to reduce friction while maintaining strong quality controls.

AI-based translation app

Problem: Localizing a website is costly, time-consuming, and potentially error-prone.

Solution: I designed and implemented an LLM-powered app that integrates with content management systems and provides inexpensive, instantaneous, and high-quality translations.

Impact:

  • Reduced translation costs by over 99% and turnaround time from days or weeks to minutes.
  • Gave our team more oversight of translation quality and eliminated gaps in translated content.
  • Enabled product, marketing, and CX teams to reuse the workflow across multiple content surfaces.

Details:

  • Integration with Zendesk and any major LLM
  • Bulk translation of multiple articles
  • Targeted translation of individual articles with preview before publishing
  • Partial or full translation based on changed sections
  • Ad-hoc translation of short texts
  • Integration with an in-house glossary

Living article management spreadsheet

Problem: As a blog or support website grows, it can be difficult to keep track of pages and other associated information like page locations, labels, and authors.

Solution: I created a script that pulls metadata for all pages on a Zendesk-based site automatically into a shared team spreadsheet.

Impact:

  • Saved hours of manual work each month.
  • Eliminated gaps and errors in article metadata.
  • Built foundation for additional automation such as broken link checks and macro tracking.

Details: The script exports article metadata from Zendesk API at regular intervals, reformats the data, and imports it to a shared spreadsheet on Google Sheets. You can sort or filter the spreadsheet by article creation date, category, section, label, and so on. This allows you to efficiently manage your content.

Broken link finder

Problem: Even if careful link checks are built into the content management process, broken links can occur for all kinds of reasons.

Solution: I created a script that checks automatically for broken links and exports the link info to a shared spreadsheet.

Impact:

  • Reduced time-to-discovery from days or weeks to less than 24 hours.
  • Improved user experience by eliminating broken links on the site.
  • Provided a simple way for non-technical team members to identify and fix broken links.

Details: The script checks all links in published articles and exports info regarding broken links to a shared spreadsheet in Google Sheets. The spreadsheet displays the number of broken links found in each article for each locale as well as the error code encountered.

The script is intended to return some false positives. For example, if the target page of a link requires authentication, the link still appears on the list. In this situation, you can manually select the ignore box in the spreadsheet. If the same link and error code appear the next time you run the script, the ignore box remains selected.
